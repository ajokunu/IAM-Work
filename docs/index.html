Saviynt x CrowdStrike Integration Implementation

Project Overview & Features

This document outlines the implementation details for integrating Saviynt Enterprise Identity Cloud (EIC) with CrowdStrike, using Okta as the Identity Provider (IdP). The integration aims to create a comprehensive identity and access management solution, combining Okta's identity management, Saviynt's governance capabilities, and CrowdStrike's cybersecurity platform.

Resources

  • Saviynt API Key (obtain from Saviynt admin portal)
  • CrowdStrike API Client ID and Secret (generate in CrowdStrike Falcon platform)
  • Okta API Token (generate in Okta admin console)
  • Saviynt EIC instance URL
  • CrowdStrike API endpoint
  • Okta org URL

Assumptions & Considerations

  • Implement API rate limiting in the integration code to respect CrowdStrike's limits.
  • Use UTC for all timestamp fields to ensure consistency.
  • Implement payload size checks before making API calls to CrowdStrike.

Error Handling & Responses

Implementation of error handling:

  1. Create a custom error logging module in Saviynt EIC.
  2. Implement exponential backoff for retries.
  3. Set up email notifications for critical errors.

Example of exponential backoff implementation:


import time

def exponential_backoff(max_retries, base_delay=1):
    for n in range(0, max_retries):
        yield base_delay * (2 ** n)

def retry_operation(operation, max_retries=3):
    for delay in exponential_backoff(max_retries):
        try:
            return operation()
        except Exception as e:
            time.sleep(delay)
    raise Exception("Max retries reached")

FAQs

Q: How to troubleshoot synchronization issues?

A: Check Saviynt EIC logs, verify API credentials, and ensure network connectivity between systems.

Q: How to add new fields to the integration?

A: Update the field mappings in Saviynt EIC connector configuration and modify the corresponding API calls.
