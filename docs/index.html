Saviynt, Okta, and CrowdStrike Integration

Project Overview

This document outlines the implementation details for integrating Saviynt Enterprise Identity Cloud (EIC) with CrowdStrike, using Okta as the Identity Provider (IdP). The integration creates a comprehensive identity and access management solution, combining Okta's identity management, Saviynt's governance capabilities, and CrowdStrike's cybersecurity platform.

Integration Architecture

Data Flows and Field Mapping

User Provisioning: Okta to Saviynt EIC

Okta Field      | Saviynt EIC Field
--------------- | ------------------
id              | externalId
userName        | userName
firstName       | firstName
lastName        | lastName
email           | email
active          | status

User Provisioning: Okta to CrowdStrike (via SCIM)

Okta Field      | CrowdStrike Field
--------------- | ------------------
userName        | uid
firstName       | firstName
lastName        | lastName
email           | email
active          | status

Access Rights Provisioning: Saviynt EIC to CrowdStrike

Saviynt EIC Field | CrowdStrike Field
------------------ | ------------------
accountID          | user_uuid
entitlementID      | roleIds

Deployment Guide

Follow these steps to integrate Saviynt, Okta, and CrowdStrike to achieve the interconnected design shown in the diagram:

Step 1: Set up Saviynt

  1. Configure identity sources in Saviynt
  2. Set up access policies and SoD rules
  3. Configure risk scoring and analytics settings

Step 2: Set up Okta

  1. Configure Okta Universal Directory
  2. Set up MFA policies
  3. Configure SSO for applications

Step 3: Integrate Saviynt with Okta

  1. Add Saviynt as an application in Okta
  2. Configure Okta in Saviynt's integration settings
  3. Set up policy synchronization and user activity logging

Step 4: Set up CrowdStrike

  1. Deploy CrowdStrike Falcon sensors to endpoints
  2. Configure CrowdStrike policies and detection settings

Step 5: Integrate Okta with CrowdStrike using SCIM

  1. Generate SCIM API token in CrowdStrike
  2. Add CrowdStrike as an application in Okta
  3. Configure SCIM provisioning in Okta for CrowdStrike

Step 6: Integrate Saviynt with CrowdStrike

  1. Configure CrowdStrike in Saviynt's integration settings
  2. Set up access governance policies for CrowdStrike in Saviynt
  3. Configure periodic access reviews for CrowdStrike users and roles

Step 7: Test and Verify the Integration

  1. Test user provisioning from Okta to CrowdStrike
  2. Verify access governance from Saviynt to CrowdStrike
  3. Test SSO login to CrowdStrike through Okta
  4. Verify policy enforcement from Saviynt to Okta

Implementation Considerations

  • Implement API rate limiting to respect CrowdStrike's limits
  • Use UTC for all timestamp fields to ensure consistency
  • Implement payload size checks before making API calls
  • Set up error logging and email notifications for critical errors
  • Implement exponential backoff for API request retries

Future Enhancements

  • Implement webhook listeners for real-time updates
  • Develop a custom dashboard for integration health monitoring
  • Expand the integration to support additional CrowdStrike features
